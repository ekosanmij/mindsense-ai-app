Version: 1.0

## Purpose

This document records the website implementation that presents MindSense AI to:

- prospective users
- investors
- internal/external stakeholders

The site mirrors the shipped app posture and does not claim features that are not in the current iOS build.

## Content model

The page is structured in this order:

1. Hero summary
2. Core product loop (state -> regulate -> capture -> learn)
3. Interactive main-surface explorer
4. Surface cards with screenshots
5. Investor brief
6. Stakeholder brief
7. Quality and governance evidence

## Surface mapping

Screens and website copy map directly to implemented app surfaces:

- `Today`
- `Regulate`
- `Data`
- `Settings`
- `Onboarding`

## Design alignment with app

Website styling uses the same directional cues as in-app design docs:

- cool-neutral base with signal-cool highlights
- layered raised surfaces with soft strokes
- calm motion (ease-out, no bounce-heavy motion)
- one clear CTA path per section
- trust-forward language and confidence framing
