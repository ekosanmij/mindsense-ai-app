        Trust, Safety, and Technical Posture

        Built with explicit boundaries and measurable quality gates

        Architecture posture

        • Single state container (`MindSenseStore`) coordinates routing, loop state, and analytics.
        • Local persistence via `UserDefaults` + Codable model snapshots.
        • Confidence score blends scenario baseline, data coverage, loop completion, and adherence.
        • Deterministic recommendation and metric delta engines support reproducible behavior.

        Quality gates

        • `Scripts/design_qa.sh` orchestrates contrast, screenshots, dynamic type, and latency checks.
        • `Scripts/contrast_audit.swift` validates critical token contrast combinations.
        • UI tests enforce accessibility scaling and interaction latency thresholds.
        • `testMarketingWebsiteScreenshotExport()` keeps web visuals tied to app reality.

        Safety boundaries

        • MindSense is a wellness product and not emergency or diagnostic care.
        • Crisis pathways are explicit, including US 988 support messaging.
        • Community and QA modules are implemented but hidden in production routing.
        • Current build does not claim cloud sync, HealthKit ingestion, or live billing integration.

        FAQ

        Common diligence questions

        Is MindSense a medical device or diagnostic product?

        No. MindSense is currently positioned as wellness guidance and does not replace emergency or licensed clinical care.

        What data posture is implemented in v1.0.0?

        The implemented runtime is local-first. Data is persisted on-device with no cloud sync path in this build.

        How does MindSense avoid recommendation overconfidence?

        Confidence labels and data coverage are explicit. Low coverage surfaces prompt additional check-ins before stronger conclusions.

        What should a user do during urgent crisis situations?

        Use local emergency services immediately. In the United States, call or text 988 for crisis support.
