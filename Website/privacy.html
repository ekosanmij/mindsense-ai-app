Privacy notice

Privacy

Last updated: February 26, 2026

Scope

This notice applies to the static marketing website located under `Website/` in this repository and to the information presented here about the MindSense AI v1.0.0 iOS app. It does not claim to cover any future hosted services, cloud backend, or analytics systems that are not implemented in the current build.

Website data collection (as shipped in this repo)

The website in this repository is a static HTML/CSS/JavaScript site and does not include built-in account creation, form submission, cookies for tracking, or analytics scripts by default.

  • No first-party analytics script is included in the repository website code.
  • No marketing pixels or ad trackers are included in the repository website code.
  • Links to GitHub may result in data collection by GitHub under GitHub's own policies.

If you deploy this website and add analytics, forms, or third-party widgets, update this page before publishing.

MindSense iOS app data posture (v1.0.0 as built)

The repository's as-built documentation and source code show a local-first storage model for MindSense AI v1.0.0. App data is persisted locally using `UserDefaults` and Codable models.

  • No cloud sync or multi-device account backend is implemented in-app for v1.0.0.
  • Authentication UX is `Sign in with Apple`.
  • Session values are stored locally to support app relaunches.
  • The app currently uses a deterministic demo/local signal model; docs list no real HealthKit ingestion pipeline yet.

Permissions and metadata boundaries

The app's Settings screen includes explicit copy about certain signal boundaries. For example, the meeting/call signals setting states that the app uses calendar busy windows and call-volume metadata for ranking top drivers, and does not use event titles, participant names, contact names, or message content.

This website reflects those implemented boundaries and does not expand them into unimplemented data uses.

Children, medical, and emergency use

MindSense is described in this repository as wellness support, not as a medical diagnosis or emergency response service. Users should not rely on the app or this website for emergency care.

Retention and deletion (repository website)

Because the repository website does not include forms or accounts by default, it does not maintain a user database in the shipped website code. Any logs generated by a hosting provider are controlled by that provider and deployment configuration.

Changes to this notice

Update this page whenever the website adds analytics/forms or when the app's data architecture changes (for example, if cloud sync or real health-data ingestion ships). The corresponding source and docs should be updated in the same commit.
